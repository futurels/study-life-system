# 学习计划与日常生活记录系统页面原型说明书

## 1. 文档基本信息

- **文档名称：** 学习计划与日常生活记录系统页面原型说明书
- **项目名称：** 学习计划与日常生活记录系统
- **英文名称：** Study Plan and Daily Life Record System

## 2. 编写目的

本说明书用于指导系统页面原型设计，明确各页面的功能定位、页面结构、核心组件、交互逻辑、页面跳转关系与设计要点，为后续低保真原型、高保真视觉设计和前端页面开发提供统一依据。

本说明书重点解决以下问题：

1. 系统需要有哪些页面；
2. 每个页面主要展示哪些内容；
3. 页面中的按钮、表单、列表、卡片、图表应如何布局；
4. 页面之间如何跳转；
5. 页面设计如何与需求文档、数据库设计和接口设计保持一致。

## 4. 原型设计总体原则

### 4.1 简洁直观原则

系统面向个人用户，页面应强调“轻量、清晰、好上手”，避免过多复杂操作。首页和各业务页面要让用户一眼看懂当前信息、下一步能做什么。

### 4.2 高效记录原则

由于系统的核心是“计划—执行—记录—复盘—统计”的成长闭环，因此新增、编辑、保存、切换日期等常用操作要尽量减少点击路径，方便快速记录。

### 4.3 单用户中心原则

系统第一版采用单用户模式，因此页面不需要多人协作、组织切换、角色管理等复杂入口。页面中的所有数据都围绕当前登录用户展开。

### 4.4 一致性原则

所有页面在以下方面保持统一：

- 顶部导航风格统一
- 页面标题区样式统一
- 表单标签风格统一
- 列表、卡片、弹窗操作位置统一
- 状态标签颜色统一
- 时间与日期展示格式统一
- 成功、失败提示风格统一

### 4.5 可扩展原则

虽然第一版只实现核心页面，但布局上应为后续扩展预留空间，例如：

- 学习分类
- 图片上传
- 番茄钟
- 提醒通知
- AI 复盘建议

## 5. 页面结构总体规划

## 5.1 页面结构模式建议

建议前端采用“登录前页面 + 登录后主框架页面”的结构。

### （1）登录前页面

- 登录页
- 注册页

此类页面不显示系统主导航栏，采用居中卡片式布局，更聚焦表单操作。

### （2）登录后页面

建议采用以下后台管理式布局：

- **顶部区域：** 系统名称、当前日期、用户头像、退出登录
- **左侧导航栏：** 首页、学习计划、生活记录、每日复盘、统计分析、个人中心
- **主内容区：** 当前页面主要功能展示区域

该结构适合前后端分离项目开发，也方便后续扩展模块。

## 6. 页面跳转关系说明

系统主流程建议如下：

1. 用户进入登录页；
2. 未注册用户可跳转到注册页；
3. 登录成功后进入首页/仪表盘；
4. 首页可快捷跳转到学习计划、生活记录、每日复盘、统计分析；
5. 左侧菜单可切换任意业务页面；
6. 点击用户头像或右上角入口可进入个人中心；
7. 点击退出登录后返回登录页。

### 6.1 页面流转关系图（文字版）

```text
登录页 <--> 注册页
   |
   v
首页/仪表盘
   |----> 学习计划页
   |----> 生活记录页
   |----> 每日复盘页
   |----> 统计分析页
   |----> 个人中心页
```

## 7. 登录页原型说明

## 7.1 页面定位

登录页用于用户输入用户名和密码进入系统，是用户访问系统的第一入口。

## 7.2 页面布局建议

建议采用左右分栏或居中卡片布局。

### 方案一：居中卡片式

- 页面背景：浅色纯色或渐变背景
- 中间卡片：
  - 系统名称
  - 系统简介
  - 用户名输入框
  - 密码输入框
  - 登录按钮
  - 跳转注册链接

### 方案二：左图右表单式

- 左侧：系统介绍文案或插画
- 右侧：登录表单卡片

第一版更推荐 **居中卡片式**，实现简单，视觉也更干净。

## 7.3 页面元素说明

| 元素 | 说明 |
|---|---|
| 系统标题 | 显示“学习计划与日常生活记录系统” |
| 用户名输入框 | 输入登录用户名 |
| 密码输入框 | 输入密码 |
| 登录按钮 | 点击后提交登录请求 |
| 注册跳转链接 | 没有账号时跳转注册页 |
| 错误提示区域 | 登录失败时显示用户名或密码错误等提示 |

## 7.4 交互说明

1. 用户输入用户名和密码；
2. 点击登录按钮；
3. 前端校验非空；
4. 调用登录接口；
5. 登录成功后保存 token；
6. 跳转到首页；
7. 登录失败时在表单下方提示错误信息。

## 8. 注册页原型说明

## 8.1 页面定位

注册页用于创建新用户账号，为系统引入用户入口。

## 8.2 页面布局建议

整体风格与登录页保持一致，采用居中卡片布局。

## 8.3 页面元素说明

| 元素 | 说明 |
|---|---|
| 用户名输入框 | 输入注册用户名 |
| 昵称输入框 | 输入展示昵称 |
| 邮箱输入框 | 可选填写 |
| 密码输入框 | 输入密码 |
| 确认密码输入框 | 再次输入密码 |

2. 密码和确认密码不一致时即时提示；
3. 注册成功后提示“注册成功，请登录”；
4. 点击确认后跳转登录页。

## 9.1 页面定位

该页面不是单独业务页，而是所有登录后页面共用的基础框架，用于提供导航、统一布局和全局操作入口。

## 9.2 页面布局建议

```text
--------------------------------------------------------------
| 系统Logo | 系统名称 | 当前日期 |        用户头像/退出登录   |
--------------------------------------------------------------
| 左侧导航栏 |                                          |
| 首页       |                                          |
| 学习计划   |              主内容区域                  |
| 生活记录   |                                          |
| 每日复盘   |                                          |
| 统计分析   |                                          |
| 个人中心   |                                          |
--------------------------------------------------------------
```

## 9.3 框架元素说明

| 区域 | 说明 |
|---|---|
| 顶部栏 | 展示系统名称、日期、用户信息、退出按钮 |
| 左侧导航 | 用于模块切换 |
| 面包屑区 | 显示当前页面路径，可选 |

## 10. 首页 / 仪表盘原型说明

## 10.1 页面定位

首页是用户登录后的第一个业务页面，用于展示今日任务概览、近期记录情况和快捷入口，是整个系统的总览中心。

## 10.2 页面目标

首页应快速回答用户三个问题：

1. 今天有什么任务？
2. 最近完成得怎么样？
3. 我下一步最适合做什么？

## 10.3 页面布局建议

建议采用“顶部统计卡片 + 中部任务概览 + 底部快捷入口”的布局。

## 10.4 页面区域划分

### （1）欢迎区

显示欢迎文案，例如：

- 你好，樱
- 今天也要认真生活

### （2）统计卡片区

建议放置 4 个统计卡片：

- 今日任务数
- 今日已完成任务数
- 本周完成率
- 本月记录天数

### （3）今日计划概览区

展示今天的学习计划列表，重点字段：

- 标题
- 时间
- 优先级
- 状态
- 快速完成按钮

### （4）今日记录提醒区

显示：

- 今天是否已写生活记录
- 今天是否已写每日复盘

若未填写，则显示快捷新增按钮。

### （5）快捷操作区

建议按钮：

- 新建学习计划
- 新建生活记录
- 新建每日复盘
- 查看统计分析

## 10.5 原型草图（文字版）

--------------------------------------------------------------

## 10.6 交互说明

- 点击统计卡片可跳转统计分析页对应模块；
- 点击“完成”按钮可快速更新任务状态；
- 点击“去记录”“去复盘”跳转对应新增表单；
- 快捷入口支持一键跳转。

## 11. 学习计划页原型说明

## 11.1 页面定位

学习计划页是系统核心业务页面，用于管理用户的学习任务，包括新增、筛选、查看、编辑、删除和状态更新。

## 11.2 页面布局建议

建议采用“查询筛选区 + 操作区 + 列表区 + 弹窗表单”的结构。

## 11.3 页面区域说明

### （1）页面标题区

- 标题：学习计划
- 副标题：管理每日学习任务与完成状态

### （2）筛选区

建议包含：

- 日期选择
- 日期范围选择
- 状态筛选
- 优先级筛选
- 查询按钮
- 重置按钮

### （3）操作区

- 新建学习计划按钮

### （4）列表区

表格列建议如下：

| 列名 | 说明 |
|---|---|
| 计划标题 | 学习计划标题 |
| 计划日期 | 所属日期 |
| 开始时间 | 开始时间 |
| 结束时间 | 结束时间 |
| 优先级 | 高 / 中 / 低 |
| 状态 | 未开始 / 进行中 / 已完成 / 已取消 |
| 备注 | 简短显示 |
| 操作 | 查看 / 编辑 / 删除 / 完成 |

### （5）新增/编辑弹窗

建议使用弹窗或抽屉形式，字段包括：

- 计划标题
- 计划内容
- 所属日期
- 开始时间
- 结束时间
- 优先级
- 备注

## 11.4 原型草图（文字版）

```text
--------------------------------------------------------------
| 学习计划                                    [新建学习计划] |
--------------------------------------------------------------
| 日期[____] 日期范围[____ ~ ____] 状态[____] 优先级[____]  |
| [查询] [重置]                                             |
--------------------------------------------------------------
| 标题           日期         时间         优先级 状态 操作  |
| Java 集合      2026-04-03   19-21        高     未完成 ... |
| SQL 题         2026-04-03   21-22        中     已完成 ... |
--------------------------------------------------------------
|                 分页区域                                   |
--------------------------------------------------------------
```

## 11.5 交互说明

1. 点击“新建学习计划”打开新增弹窗；
2. 点击“编辑”打开编辑弹窗并回填数据；
3. 点击“删除”弹出确认提示；
4. 点击“完成”直接将状态更新为已完成；
5. 点击行内“查看”可打开详情弹窗或详情侧边栏。

## 11.6 设计要点

- 高优先级任务可使用醒目颜色标签；
- 已完成任务可弱化显示；
- 筛选区尽量简洁，避免过多条件；
- 新增和编辑尽量复用一套表单组件。

## 12. 生活记录页原型说明

## 12.1 页面定位

生活记录页用于记录和查看用户每天的生活日志，是系统中偏“日记型”的页面。

## 12.2 页面布局建议

建议采用“左侧日期/记录列表 + 右侧内容区”的布局，也可以采用“顶部日期筛选 + 卡片列表”的布局。

第一版推荐：**顶部日期筛选 + 卡片列表 + 编辑弹窗**，实现相对简单。

## 12.3 页面区域说明

### （1）页面标题区

- 标题：生活记录
- 副标题：记录每天的生活状态与感受

### （2）筛选区

建议包含：

- 日期选择
- 日期范围
- 心情状态筛选
- 查询 / 重置按钮

### （3）操作区

- 新建生活记录按钮

### （4）记录列表区

建议用卡片式展示，每张卡片显示：

- 日期
- 心情标签
- 睡眠时长
- 日记摘要
- 饮食简述
- 运动简述
- 操作按钮

### （5）新增 / 编辑表单

字段建议包括：

- 记录日期
- 日记正文
- 心情状态
- 睡眠时长
- 饮食简述
- 运动简述

## 12.4 原型草图（文字版）

```text
--------------------------------------------------------------
| 生活记录                                    [新建记录]     |
--------------------------------------------------------------
| 日期[____] 日期范围[____ ~ ____] 心情[____] [查询] [重置] |
--------------------------------------------------------------
| 2026-04-03   平静   睡眠7.5h                               |
| 今天上午状态一般，下午完成了学习任务，晚上去散步了……       |
| 饮食：早餐简单，晚饭较清淡                                 |
| 运动：散步30分钟                                           |
| [查看] [编辑] [删除]                                       |
--------------------------------------------------------------
| 2026-04-02   开心   睡眠8.0h                               |
| ……                                                         |
--------------------------------------------------------------
```

## 12.5 交互说明

- 若当天已存在记录，则点击“新建记录”时提示用户直接编辑；
- 点击“查看”可打开详情弹窗；
- 点击“编辑”进入编辑弹窗；
- 删除操作需二次确认；
- 列表摘要建议自动截断，详情页再展示完整内容。

## 12.6 设计要点

- 页面应更偏“内容阅读型”，不要做得太像纯后台表格；
- 心情、睡眠、运动等信息可标签化展示；
- 文字区留白要足，提升阅读体验。

## 13. 每日复盘页原型说明

## 13.1 页面定位

每日复盘页用于对一天进行总结，是系统“成长管理”价值最强的页面之一。

## 13.2 页面布局建议

建议采用“日期筛选 + 复盘卡片 / 列表 + 表单编辑区”的结构。

## 13.3 页面区域说明

### （1）页面标题区

- 标题：每日复盘
- 副标题：总结今天，规划明天

### （2）筛选区

- 指定日期
- 日期范围
- 查询 / 重置按钮

### （3）操作区

- 新建每日复盘按钮

### （4）复盘列表区

建议卡片化展示，核心字段：

- 复盘日期
- 今日完成事项摘要
- 今日未完成事项摘要
- 今日整体评价

### （5）新增 / 编辑表单

字段包括：

- 评分可采用滑块、单选按钮或数字输入框；
- 完整内容可在详情弹窗中查看。

## 13.6 设计要点

- 复盘页应体现“回顾 + 反思 + 计划”的层次；
- 与生活记录页区分开，语气和布局更偏总结型；
- 今日评分建议视觉化，如星级、进度条、数字徽章。

## 14. 统计分析页原型说明

## 14.1 页面定位

统计分析页用于展示任务完成情况、记录频率和趋势变化，是系统可视化展示的核心页面。

## 14.2 页面布局建议

建议采用“筛选区 + 指标卡片 + 图表区”的布局。

## 14.3 页面区域说明

### （1）页面标题区

- 标题：统计分析
- 副标题：查看近期任务与记录趋势

### （2）时间范围切换区

建议支持：

- 本周
- 本月
- 最近 7 天
- 最近 30 天

### （3）核心指标卡片区

建议指标：

- 本周任务总数
- 本周已完成任务数
- 本周完成率
- 本月生活记录天数
- 本月复盘天数

### （4）图表展示区

建议包含：

1. 最近 7 天任务完成趋势图（折线图）
2. 本周任务状态分布图（饼图或柱状图）
3. 本月生活记录 / 复盘天数统计图（柱状图）

## 14.5 交互说明

- 时间按钮切换后刷新指标与图表；
- 卡片数值和图表联动；
- 若暂无数据，应显示空状态而不是报错；
- 图表下方可显示简短说明文字。

## 14.6 设计要点

- 第一版统计不要堆太多图表；
- 重点突出“完成率、记录天数、趋势”三个概念；
- 图表标题要直白，避免太学术化。

## 16. 弹窗与通用组件原型说明

为了提高页面一致性，建议统一设计以下通用组件。

## 16.1 确认弹窗

适用于删除、退出登录等高风险操作。

### 示例文案

- 是否确认删除该条学习计划？
- 是否确认删除该条生活记录？
- 是否确认退出登录？

## 16.2 空状态组件

当页面没有数据时统一显示：

- 图标
- 提示文案
- 引导按钮

例如：

- 还没有学习计划，马上创建第一条任务吧
- 今天还没有生活记录，去写一条吧
- 当前暂无统计数据

## 16.3 状态标签组件

统一用于展示：

- 学习计划状态
- 优先级
- 心情状态

建议示例：

- 高优先级：红色标签
- 中优先级：橙色标签
- 低优先级：蓝色标签
- 已完成：绿色标签
- 未开始：灰色标签

## 16.4 日期选择组件

学习计划、生活记录、每日复盘、统计分析页都会用到，应统一样式和交互规则。

## 19. 原型绘制建议

如果你后续要画原型，建议按以下顺序进行：

1. 先画登录页和注册页；
2. 再画登录后统一框架；
3. 再画首页；
4. 接着画学习计划页、生活记录页、每日复盘页；
5. 最后补统计分析页和个人中心页；
6. 再补弹窗、空状态、删除确认框等通用组件。

### 建议原型工具

- Axure RP
- 墨刀
- Figma
- ProcessOn（简单流程辅助）

### 建议原型层级

- **低保真原型：** 先确定布局、模块、按钮位置
- **中保真原型：** 补充字段、状态、提示信息
- **高保真设计：** 再统一颜色、图标、字体、视觉风格
