## 3. 项目背景

随着学习任务增多和生活节奏加快，个人在日常生活中常常面临计划不清晰、执行不连续、复盘不及时、生活记录零散等问题。传统的纸质记录方式不便于长期保存与统计分析，而普通记事工具又难以将“学习计划”“生活记录”“每日复盘”进行统一管理。

因此，开发一个集**学习计划管理、生活日志记录、每日复盘总结、数据统计分析**于一体的个人管理系统，具有较强的实用价值。该系统能够帮助用户合理安排学习任务，记录日常生活状态，及时进行阶段性总结与反思，从而提升个人的自律能力、学习效率与生活管理能力。

------

## 4. 项目建设目标

本系统拟构建一个面向个人用户的前后端分离管理平台，实现对学习计划和日常生活内容的统一记录与管理。系统建设目标如下：

1. 实现用户注册、登录与个人信息管理功能。
2. 实现每日学习计划的创建、修改、删除、查询与完成状态管理。
3. 实现日常生活记录功能，便于用户对生活状态、心情、习惯进行记录。
4. 实现每日复盘功能，帮助用户总结完成情况并制定次日计划。
5. 实现统计分析功能，为用户提供任务完成情况、记录频率等可视化信息。
6. 采用前后端分离架构，提高系统的可维护性、扩展性与开发规范性。

------

## 5. 系统目标用户

本系统的主要服务对象为有学习规划和生活管理需求的个人用户，尤其适用于：

- 在校学生
- 考证备考人员
- 需要进行自我管理的人群
- 希望长期记录成长过程的用户

------

## 6. 系统业务范围

本系统围绕“计划—执行—记录—复盘—统计”这一闭环展开，核心业务范围包括：

- 用户账户管理
- 学习计划管理
- 日常生活记录管理
- 每日复盘总结管理
- 数据统计与可视化展示

系统不涉及多人协作、复杂审批流、即时通讯等大型企业级业务，定位为**轻量级个人成长管理系统**。

## 7. 角色分析

本系统第一版建议采用单角色设计，即：

### 7.1 普通用户

普通用户是系统的唯一核心使用者，主要功能包括：

- 注册和登录系统
- 管理自己的学习计划
- 记录自己的生活内容
- 填写每日复盘
- 查看个人统计信息
- 修改个人资料

### 7.2 管理员（可选）

第一版其实可以不做管理员。
 如果后续为了项目完整度想扩展，可以增加管理员角色，用于：

- 管理用户信息
- 查看系统运行数据
- 管理公告或分类信息

但从你的项目主题来看，**第一阶段不建议强行加入管理员模块**，否则会增加开发复杂度。

## 8. 功能需求分析

## 8.1 用户模块需求

用户模块负责用户身份认证与个人资料管理，是系统运行的基础模块。

### 功能需求

1. 用户注册
2. 用户登录
3. 用户退出登录
4. 查询个人信息
5. 修改个人资料
6. 修改密码

### 功能说明

- 用户可以使用用户名和密码完成注册。
- 登录成功后，系统返回身份凭证，用于后续接口访问。
- 用户只能查看和修改自己的个人信息，不能操作他人数据。

------

## 8.2 学习计划模块需求

学习计划模块是本系统的核心功能模块，用于帮助用户安排和管理每日学习任务。

### 功能需求

1. 新建学习计划
2. 编辑学习计划
3. 删除学习计划
4. 查看学习计划详情
5. 查询某日/某段时间内的学习计划
6. 标记学习计划完成状态
7. 按优先级、状态、日期筛选学习计划

### 功能说明

学习计划一般包含以下信息：

- 计划标题
- 计划内容
- 所属日期
- 开始时间
- 结束时间
- 优先级
- 完成状态
- 创建时间

### 示例场景

用户可以创建“今天学习 Java 集合 2 小时”“刷 3 道 SQL 题”“背 20 个英语单词”等任务，并在完成后手动标记状态。

------

## 8.3 日常生活记录模块需求

生活记录模块用于记录用户每天的日常状态，使系统不仅能管理学习，还能保留生活轨迹。

### 功能需求

1. 新建生活记录
2. 编辑生活记录
3. 删除生活记录
4. 查看生活记录详情
5. 按日期查询生活记录
6. 记录心情、睡眠、运动等扩展信息

### 功能说明

生活记录可以包含：

- 记录日期
- 日记正文
- 心情状态
- 睡眠时长
- 饮食/运动简述
- 创建时间

### 示例场景

用户可以记录“今天起床较晚，上午状态一般，下午完成了学习任务，晚上去散步了”等内容，用于形成连续的日常日志。

------

## 8.4 每日复盘模块需求

每日复盘模块用于用户对一天的学习和生活进行总结，是体现系统“成长管理”特点的重要功能。

### 功能需求

1. 新建每日复盘
2. 编辑每日复盘
3. 删除每日复盘
4. 查看复盘详情
5. 按日期查询复盘记录

### 复盘内容建议

- 今日完成事项
- 今日未完成事项
- 未完成原因
- 今日整体评价
- 明日计划
- 今日评分

### 功能价值

通过每日复盘，用户可以形成长期总结习惯，从单纯“记任务”提升为“记录成长过程”。

------

## 8.5 数据统计模块需求

统计模块主要用于展示用户在某段时间内的学习与记录情况，增强系统的可视化和实用性。

### 功能需求

1. 统计本周学习任务总数
2. 统计本周已完成任务数
3. 统计任务完成率
4. 统计本月生活记录天数
5. 统计本月复盘天数
6. 展示最近一周或一月的数据趋势

### 功能说明

第一版统计不需要过于复杂，建议以基础图表和数字展示为主，例如：

- 本周完成任务数
- 本月记录天数
- 今日任务完成率
- 最近 7 天任务趋势图

------

## 9. 非功能需求分析

除功能需求外，系统还应满足一定的非功能需求。

### 9.1 性能需求

1. 系统页面响应时间应尽量控制在 3 秒以内。
2. 普通查询操作应快速返回。
3. 在个人单用户场景下应保持流畅运行。

### 9.2 安全需求

1. 用户密码需要加密存储，不能明文保存。
2. 登录后通过 JWT 或 Session 进行身份校验。
3. 用户只能访问自己的数据，不能查看其他用户的学习计划和生活记录。
4. 输入内容需进行基本校验，防止非法请求。

### 9.3 可维护性需求

1. 前后端代码分离，模块职责清晰。
2. 后端采用分层架构，便于后续功能扩展。
3. 前端按页面、接口、状态管理分类组织代码。
4. 统一返回格式和异常处理机制。

### 9.4 可扩展性需求

系统后续应支持扩展以下能力：

- 学习分类管理
- 番茄钟功能
- 提醒通知
- 图片上传
- 数据导出
- AI 自动生成复盘总结

### 9.5 易用性需求

1. 页面布局应简洁直观。
2. 常用操作应尽量减少点击步骤。
3. 记录与新增功能应方便快捷。

## 10. 业务流程分析

## 10.1 用户使用主流程

1. 用户进入系统登录页
2. 注册或登录系统
3. 进入首页查看今日计划与概览信息
4. 创建或编辑学习计划
5. 记录当天生活内容
6. 晚上填写每日复盘
7. 查看统计数据，了解近期完成情况

------

## 10.2 学习计划业务流程

1. 用户新增学习计划
2. 系统保存计划数据
3. 用户在计划列表中查看当日任务
4. 用户完成任务后修改状态
5. 系统更新完成情况并参与统计分析

------

## 10.3 生活记录业务流程

1. 用户进入生活记录页面
2. 选择日期并填写内容
3. 系统保存记录
4. 用户后续可以查看、修改或删除历史记录

------

## 10.4 每日复盘业务流程

1. 用户进入每日复盘页面
2. 填写完成情况、未完成原因、明日计划等内容
3. 系统保存复盘记录
4. 复盘数据纳入月度和周度统计展示

------

## 11. 前后端需求划分

为了体现前后端分离项目特点，需求分析中可以明确区分前端职责与后端职责。

## 11.1 前端需求

前端主要负责页面展示、用户交互、表单校验、数据请求与图表展示。

### 前端主要页面

- 登录页
- 注册页
- 首页/仪表盘
- 学习计划页
- 生活记录页
- 每日复盘页
- 统计分析页
- 个人中心页

### 前端主要职责

1. 页面布局与交互设计
2. 表单数据提交与校验
3. 调用后端接口并展示数据
4. 保存登录状态
5. 实现列表、详情、新增、编辑、删除等操作
6. 实现图表和统计信息展示

------

## 11.2 后端需求

后端主要负责业务逻辑处理、数据存储、权限控制、接口提供与异常处理。

### 后端主要职责

1. 用户注册登录逻辑
2. JWT 身份校验
3. 学习计划数据管理
4. 生活记录数据管理
5. 每日复盘数据管理
6. 数据统计逻辑实现
7. 统一返回体与统一异常处理
8. 数据访问与数据库操作
